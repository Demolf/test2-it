# Новости на Node.js с использованием Express, EJS и MySQL

Это веб-приложение для управления новостями, построенное на Node.js, Express и EJS. Оно позволяет просматривать список новостей и добавлять новые через API. Все данные хранятся в базе данных MySQL.

## Описание проекта

Приложение состоит из двух частей:

1. **Фронтенд на EJS** — отображает список новостей на главной странице.
2. **API на Express** — предоставляет эндпоинт `/api/news` для получения и добавления новостей.
3. **База данных MySQL** — хранит все новости (заголовок, содержание, дата).

### Как это работает

1. При запуске сервера автоматически создаётся база данных `test` и таблица `news`, если их ещё нет.
2. Главная страница (`/`) отображает список новостей с помощью шаблонизатора EJS.
3. API (`/api/news`) позволяет получать и добавлять новости в формате JSON.
4. Все данные сохраняются в MySQL и доступны после перезагрузки сервера.

### Используемые технологии

- **Node.js** — сервер на JavaScript.
- **Express** — веб-фреймворк для маршрутизации и обработки запросов.
- **EJS** — шаблонизатор для генерации HTML на сервере.
- **MySQL** — реляционная база данных для хранения новостей.
- **dotenv** — загрузка переменных окружения из `.env`.

## Установка и запуск

1. Убедитесь, что у вас устаовлены [Node.js](https://nodejs.org/) и [MySQL](https://www.mysql.com/).
2. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Demolf/test-it.git
   cd test-it
   ```
3. Установите зависимости:
   ```bash
   npm install
   ```
4. Настройте переменные окружения (создайте файл `.env`):
   ```env
   PORT=3000
   DB_HOST=localhost
   DB_PORT=3306
   DB_USER=root
   DB_PASSWORD=
   ```
   Или измените значения прямо в `db/config.js`.
5. Запустите сервер:

   ```bash
   npm run start
   ```

   — при первом запуске база данных `test` и таблица `news` создадутся автоматически.

6. Откройте в браузере:
   ```
   http://localhost:3000
   ```

## Структура проекта

- `index.js` — основной сервер.
- `db/config.js` — подключение и инициализация базы данных.
- `routes/` — маршруты приложения.
- `views/` — шаблоны EJS.
- `public/` — статические файлы (CSS, JS, Images).
- `package.json` — зависимости и скрипты.
